<ROLE>
  Act as a principal solutions architect, senior full-stack engineer, senior product designer, Primavera P6 integration architect, scheduling domain expert, and AI workflow designer.
  Design and generate a production-oriented web application for construction project planning and schedule generation.
  The system must be chat-first, AI-native, visually rich, editable, modular, and architected for future enterprise expansion.
</ROLE>

<CORE_OBJECTIVE>
  Build a web application where a user can describe a construction or industrial project in natural language through a chat interface, and the system:
  1. collects and structures project data,
  2. asks clarifying questions,
  3. generates a project WBS,
  4. generates tasks and milestones,
  5. proposes durations, logic links, and sequencing,
  6. validates the schedule structure,
  7. visualizes the result in multiple views,
  8. allows full interactive editing,
  9. compares the result against historical and template-based knowledge,
  10. supports Primavera P6 synchronization from the start of the solution design.
</CORE_OBJECTIVE>

<IMPORTANT_BUSINESS_CONTEXT>
  The target domain is large construction and industrial projects, including EPC, EPCM, multi-contract industrial construction, mining, oil and gas, process plants, utilities, and infrastructure.
  The system is not just a generic task manager. It is a planning and schedule engineering platform.
  It must be designed around construction planning logic, engineering deliverables, procurement workflows, construction sequencing, and schedule quality.
</IMPORTANT_BUSINESS_CONTEXT>

<PRODUCT_VISION>
  The application should feel like a combination of:
  - AI planning copilot,
  - schedule builder,
  - interactive Gantt editor,
  - network logic editor,
  - Kanban execution board,
  - template-driven construction planning system,
  - future-ready Primavera-connected planning workspace.
</PRODUCT_VISION>

    <SERVICE name="chat-orchestration-service">
      Handles user prompts, clarifying questions, structured extraction of project data, and generation workflow orchestration.
    </SERVICE>

    <SERVICE name="project-configuration-service">
      Stores project metadata, project type, units, zones, scope assumptions, contract strategy, execution model, calendars, and settings.
    </SERVICE>

    <SERVICE name="template-library-service">
      Stores preloaded templates for engineering, procurement, construction, commissioning, infrastructure, and industrial project structures.
    </SERVICE>

    <SERVICE name="schedule-engine-service">
      Generates WBS, activities, milestones, durations, logic, calendars, and sequencing rules.
    </SERVICE>

    <SERVICE name="dependency-management-service">
      Manages logic links, relationship types, lags, lead/lag editing, validation, and graph integrity.
    </SERVICE>

    <SERVICE name="validation-service">
      Detects cycles, orphan tasks, missing predecessors, missing successors, inconsistent logic, unrealistic durations, broken hierarchy, and schedule quality issues.
    </SERVICE>

    <SERVICE name="ai-recommendation-service">
      Evaluates new tasks and edits using templates, historical patterns, and rule-based / AI reasoning. Suggests improvements rather than forcing them.
    </SERVICE>

    <SERVICE name="historical-comparison-service">
      Matches new tasks or project structures with prior project patterns and standard libraries.
    </SERVICE>

    <SERVICE name="visualization-state-service">
      Ensures that Gantt, Kanban, and Network views all reflect one synchronized underlying schedule model.
    </SERVICE>

    <SERVICE name="p6-integration-service">
      Handles Primavera-aligned schema translation, import/export mapping, and later direct synchronization support.
    </SERVICE>

    <SERVICE name="audit-and-change-service">
      Tracks user edits, AI suggestions accepted or rejected, version history, and schedule change log.
    </SERVICE>
  </SERVICES>
